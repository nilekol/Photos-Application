LoginController Documentation

The LoginController class is responsible for logging in the users. Additionally, if a user does not have an account, there is functionality for signing a user up. The admin is able to access admin functionality in this class. Author: Nile Kolenovic

Fields

stage

The stage used for managing scenes.

root

The root element for the scene.

idAndPasswords

An instance of IdAndPasswords used for retrieving data from a file.

FXML Elements

loginStatusLabel

A label for displaying login status.

createAccountButton

The button for creating a new user account.

loginButton

The button for logging in.

signUpButton

The button for signing up a new user.

passwordField

The text field for entering the user's password.

Methods

signUpButtonClick(ActionEvent event)

Handles the event when a user clicks the "Sign Up" button. The method gets the text from the usernameField and passwordField. If a user does not exist with such username and password, a new user is created and is written to the loginInfo.ser file. When the sign up is successful, the "Sign up success" is displayed. Otherwise, "Username already exists" is displayed.

Parameters: ActionEvent event - the action event triggered by the user

Author: Nile Kolenovic

loginButtonClick(ActionEvent event)

Handles the event when a user clicks the "Login" button. The method gets the text from the usernameField and passwordField. If a user exists with such username and password, the user is logged in. Otherwise, "Login failed" is displayed. If the username and password are admin, change the scene to the Administrator scene.

Parameters: ActionEvent event - the action event triggered by the user

Throws: IOException - if there is an error during the login process

Author: Nile Kolenovic
