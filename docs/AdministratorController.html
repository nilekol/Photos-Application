AdministratorController Documentation

The AdministratorController class is responsible for managing the administrative functions of the application. Has functionality for adding users, deleting users, and listing all users. This class also interacts with the IdAndPasswords class to read and write user login information and photo albums. This ensures data persistence between sessions. @author Nile Kolenovic

Fields

stage

The stage used for managing scenes.

scene

The current scene.

idAndPasswords

An instance of IdAndPasswords used for retrieving data from a file.

loginInfo

A hashmap of usernames and their respective passwords.

photoAlbums

A hashmap of usernames and their respective photo albums.

FXML Elements

addPasswordField

The password field for adding a new user.

Methods

addUserButtonClick(ActionEvent event)

Handles the action when the admin clicks the "Add User" button. Admin is able to set the new user's username and password by typing in the respective text fields. Once this method is called, the "Add User" button changes to "Confirm". If the admin clicks the "Confirm" button, the new user is added. If the username already exists, the user is not added. This method also interacts with the IdAndPasswords class to write the new user to the file.

Parameters: ActionEvent event - the action event triggered by the user

Author: Nile Kolenovic

deleteUserButtonClick(ActionEvent event)

Handles the action when the admin clicks the "Delete User" button. Admin is able to delete a user by typing in the username of the user in the respective text field. Once this method is called, the "Delete User" button changes to "Confirm". If the admin clicks the "Confirm" button, the user is deleted. If the user does not exist, the text "User [X] does not exist" is displayed in the listAllUsersTextField. If the user is the admin or stock user, the text "Cannot delete admin or stock user" is displayed in the listAllUsersTextField. This method also interacts with the IdAndPasswords class to write the new user to the file.

Parameters: ActionEvent event - the action event triggered by the user

Author: Nile Kolenovic

listAllUsersButtonClick(ActionEvent event)

Handles the action when the admin clicks the "List Users" button. Admin is able to list all users by clicking the "List Users" button. A list of all users currently in the idAndPasswords hashmap is displayed in the listAllUsersTextField.

Parameters: ActionEvent event - the action event triggered by the user

Author: Nile Kolenovic

logOutButtonClick(ActionEvent event)

Handles the action when the admin clicks the "Log Out" button. Admin is able to log out by clicking the "Log Out" button. The scene is changed to the Login scene.

Parameters: ActionEvent event - the action event triggered by the user

Author: Nile Kolenovic

Throws: IO Exception - if there is an error reading the file
